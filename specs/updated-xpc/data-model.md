# Data Model: MVP XPC Connection

This document describes the data structures passed between the main application and the XPC service for the MVP XPC connection feature.

## 1. `getTimestamp` Request

- **Direction**: `NetworkReporter` (Client) → `NetworkReporterService` (Service)
- **Payload**: None. The request is a method invocation without parameters.

## 2. `getTimestamp` Response

- **Direction**: `NetworkReporterService` (Service) → `NetworkReporter` (Client)
- **Payload**: The response is delivered via a completion handler with two optional parameters: `String?` for the timestamp and `Error?` for potential errors. One of these should be non-nil.

### Success Case

- **Type**: `String?` (non-nil)
- **Format**: An ISO 8601 timestamp (e.g., `"2025-12-24T18:30:00Z"`).
- **Error Parameter**: `nil`

### Failure Case

- **Type**: `String?` (nil)
- **Format**: Not applicable
- **Error Parameter**: `Error?` (non-nil)
- **Description**: A standard Swift `Error` object, typically an `NSError` generated by the `NSXPCConnection` framework in case of a connection failure or service error.
